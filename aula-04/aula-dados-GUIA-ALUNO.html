Central de Dados

Transforme o histórico de tráfego num painel que decide. Séries no tempo, comparação de campanhas e um board de especialistas de mercado lendo os seus números — sem tocar em nenhuma campanha na Meta.

Ver o painel de exemplo → Configurar chaves Como obter o token da Meta

O que você vai montar

O Squad de Dados conduz tudo — uma skill por função, com porta de entrada única: /analista-de-dados (hub que roda coleta → board → painel → retroalimentação; cada etapa também existe como skill própria: /coletor-de-dados, /board-de-especialistas, /painel-de-dados, /retroalimentacao, /gestor-de-campanhas). Ela configura as chaves (ou roda em Modo Exemplo), puxa os dados reais da Meta, monta o painel local no estilo Grafana/Looker com a sua identidade visual e convoca o board de especialistas.

A demo nunca trava. Sem credenciais no .env, o painel roda sobre dados de exemplo (rotulados modo exemplo). Com o token da Meta, ele lê a sua conta.

No Passo 0, a skill pergunta antes de decidir

  • Já tem .env? Ela pergunta se você quer seguir com a conta atual ou usar outra — e lista as contas que o seu token acessa para você escolher.
  • Não tem .env? Ela pergunta se você quer criar o .env e configurar a API agora (com ajuda, seguindo o tutorial de token) ou seguir no Modo Exemplo só para visualizar.

Os 4 passos

  1. Chaves / Modo — valida o .env (delegado ao /zelador) ou cai no Modo Exemplo.
  2. Dados — séries por dia, mês e quarter, comparação de períodos e de campanhas.
  3. Board — 4 clones de especialistas leem os dados e recomendam uma alavanca cada.
  4. Painel — HTML self-contained (abre offline, vira PDF) na identidade da sua marca.

O board de especialistas

Media Buyer

Lê gasto, ROAS e frequência entre campanhas. Diz onde escalar, pausar e para onde realocar verba — uma campanha por vez.

Analista de Dados

Lê as séries e a comparação de períodos. Separa tendência de ruído e avisa o que é Real vs. Estimado.

Diretor de Criativos

Lê queda de CTR, frequência e idade do criativo. Diz quando rotacionar e pede nova leva de ganchos ao Briefista.

CRO / Growth

Lê a relação clique → compra e a consistência plataforma↔checkout↔caixa. Aponta onde a venda vaza.

Cada clone ancora a leitura num número real e cita o selo. A síntese concilia as vozes e ordena as alavancas. No fim: VOCÊ REVISA — o board recomenda, você decide.

Como ler os selos

SeloSignifica
RealVeio pronto da Meta. Nada é calculado localmente.
EstimadoAtribuição da plataforma. Todo ROAS é Estimado — não é caixa confirmado.
CalculadoVariação entre dois valores Real (comparação de períodos).
não fornecidoCampo ausente — não se inventa nem se deriva. No trimestre, taxas não são recalculadas.
