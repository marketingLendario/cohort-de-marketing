Parte 0

Antes de tudo (uma vez só)

Este bloco você faz uma vez, no primeiro dia. Prepara o computador para rodar tudo o que vem depois. Não pule nada: 90% dos travamentos do curso nascem de um passo aqui esquecido. Ao final, o comando /comecar confere se está tudo no lugar.

1

O que é o terminal e como abrir

O terminal é a janelinha onde você digita comandos em vez de clicar em botões. É por ele que o Claude Code roda. Abrir é rápido:

Windows: abra o Explorador de Arquivos, clique na barra de endereço no topo, digite cmd e aperte Enter. O terminal abre já dentro da pasta que estava aberta.

Mac: abra o Spotlight (a lupa no canto, ou Cmd + espaço), digite Terminal e aperte Enter.

2

Instale o Claude Code (ou o Codex)

Baixe o Claude Code em docs.claude.com/claude-code. É um aplicativo de terminal. Se você usa o Codex, ele também funciona — a diferença é só onde a ferramenta procura as skills e se as páginas abrem sozinhas (veja o alerta no fim da Parte 2).

3

Clone o repositório do cohort (ou atualize, se já tem)

Primeira vez? No terminal, rode:

git clone https://github.com/marketingLendario/cohort-de-marketing.git
cd cohort-de-marketing

Se aparecer "git não é reconhecido", instale o Git em git-scm.com/download/win (Windows) e reabra o terminal.

Sem Git (alternativa ZIP): no GitHub, botão verde Code → Download ZIP, e descompacte. Atenção: pelo ZIP você não recebe as atualizações do curso — o git pull não funciona numa pasta baixada como ZIP. Prefira o git clone sempre que possível.

Já tem a pasta da Aula 1? Não clone de novo. Entre nela e atualize:

cd caminho/da/sua/pasta/cohort-de-marketing
git pull

No Windows, em vez de digitar o caminho: abra a pasta cohort-de-marketing no Explorador, clique na barra de endereço, digite cmd, Enter — o terminal já abre no lugar certo. Para conferir onde você está, use dir (Windows) ou ls (Mac/Linux).

4

Abra o Claude na pasta

Dentro da pasta cohort-de-marketing, rode:

claude

As skills em .claude/skills/ carregam automaticamente. No Codex, abra com codex: ele usa .agents/skills/, que é o espelho literal das mesmas skills.

5

Rode /comecar — o passo zero que confere tudo

Com o Claude aberto, digite:

comecar

É o primeiro comando do cohort e a rede de segurança do curso inteiro. Ele confere se as skills carregaram, checa a chave do Apify no .env (e ajuda a configurar se faltar), verifica Node e Python, resolve a /design-md duplicada e te entrega um único próximo passo. Se algo travar em qualquer momento — na Aula 1 ou na Aula 2 — volte para ele. (No Codex, chame com @comecar.)

Kit de sobrevivência do terminal

Esc cancela uma pergunta ou ação que o Claude deixou pendente. A seta pra cima (↑) repete o último comando. Shift+Tab liga o modo de aceitação automática (o Claude para de pedir confirmação a cada passo) — use só quando já confia no fluxo. E nunca use --dangerously-skip-permissions sem entender o risco: ele remove todas as proteções de uma vez.

Custo, modelo e sessão

As skills são roteiros prontos e rodam bem em Sonnet — não precisa de Opus pra tudo. Rodadas pesadas (coletar + gerar nos 3 formatos) consomem mais, então dá pra fazer por partes. E o mais importante: o seu progresso mora nos arquivos, não na conversa. Pode fechar ou limpar o chat entre uma skill e outra sem perder nada — pra voltar, é só abrir o Claude na pasta e rodar a próxima skill.

Como abrir as páginas .html que as skills geram

Quase toda skill entrega arquivos .html. No Claude Code eles costumam abrir sozinhos; se não abrir (ou se você usa o Codex), dê dois cliques no arquivo ou rode no terminal, conforme o seu sistema:

macOS: open caminho/do/arquivo.html
Windows: start "" caminho\do\arquivo.html
Linux: xdg-open caminho/do/arquivo.html

No Windows use barra invertida (\) nos caminhos e mantenha as aspas vazias depois de start. Para copiar arquivos no Windows o comando é copy (no Mac/Linux é cp).

Parte 1 · Aula 1

Aula 1: Pesquisa → Oferta

A base de tudo. Você roda 5 skills na ordem — cada uma alimenta a próxima — e sai com 24 arquivos: a pesquisa de avatar, os dossiês dos concorrentes, as tendências, o swipe file e o offerbook (o Livro da Oferta), que é a matéria-prima da Aula 2. Cada entregável visual sai em três formatos: .md (fonte de verdade), .html (abre no navegador) e .pdf.

A regra-mãe

Pesquisa antes da oferta, oferta antes de copy, copy antes de ads. Pular essa ordem queima verba. Toda seção com dado real precisa de citação literal (verbatim); o que não tem fonte é marcado [SUPOSIÇÃO], nunca inventado.

Antes de rodar: qual é o seu caso?

As skills funcionam para qualquer negócio. Só ajuste o ponto de partida ao seu perfil:

Não tenho projeto ainda

Sem trava. Três saídas: (1) use você mesmo como projeto (se você fosse uma empresa, como seria o seu marketing?); (2) foque a pesquisa num nicho que te interessa — o /avatar-funil acha a brecha com você; (3) olhe pro seu contexto e resolva a dor real de quem está ao seu redor.

O avatar acha o nicho com você — não precisa ter produto pra começar.

Sou agência / atendo clientes

Trabalhe um cliente por vez. Rode o ciclo completo das 5 skills para um cliente, feche o offerbook dele, e só então comece o próximo. Um cliente, uma pasta, uma conversa — não misture dois na mesma sessão.

Um cliente por vez, do avatar ao offerbook.

Sou afiliado

Você não é o dono do produto, mas ainda precisa do offerbook. Trate a oferta do produtor como a sua: registre no offerbook o que o produtor entrega (promessa, preço, garantia, bônus) e construa a sua pesquisa de avatar e ângulo em cima disso.

A oferta do produtor entra registrada no offerbook.

Skill 1 de 5

/avatar-funil

Pesquisa de avatar em 7 dimensões + focus group sintético em 3 personas testando a sua headline. Faz 4 passos: minera dor real em 3 fontes (reviews, comunidades, redes), ranqueia as dores pelos 4 critérios da "dor cara", monta o avatar ancorado na frase verbatim do lead e valida a headline nos 3 perfis antes de você gastar em mídia. Roda em modo rede (busca sozinha) ou offline (você cola o material).

avatar-funil [nome do seu negócio ou nicho]

Arquivos que saem (3):

  • relatorio-avatar.md — fonte de verdade
  • relatorio-avatar.html — versão visual (abre no navegador)
  • relatorio-avatar.pdf — para imprimir ou compartilhar

Gate antes de avançar:

  • Avatar tem nome, idade e frase verbatim do lead?
  • Headline foi validada nos 3 perfis do focus group?
  • Campos sem dado estão marcados [SUPOSIÇÃO] (e não inventados)?
Skill 2 de 5

/espiao-do-concorrente

Dossiê multi-fonte de um concorrente — rode uma vez para cada. Varre Meta Ad Library, Google Ads Transparency, TikTok Ads, redes orgânicas (Instagram, LinkedIn, X), o site (oferta, preço, garantia) e a reputação (reviews, menções). Devolve ganchos, ofertas, ângulos e, principalmente, as brechas. Prefira brecha de ângulo a brecha de preço: preço se copia em 30 dias, história não.

espiao-do-concorrente [nome ou @ do concorrente]

Arquivos que saem (3 por concorrente):

  • dossie-{concorrente}.md — 9 seções (resumo, fontes, ganchos, ofertas, ângulos, formatos, reputação, brechas, 3 jogadas recomendadas)
  • dossie-{concorrente}.html — o mesmo dossiê renderizado
  • dossie-{concorrente}.pdf — para enviar/imprimir

Gate antes de avançar:

  • Rodou para no mínimo 3 concorrentes diretos?
  • Cada dossiê tem pelo menos 10 peças analisadas (senão sinalize "amostra parcial")?
  • Cada dossiê termina com 3 jogadas acionáveis?
Configure o Apify agora — a próxima skill precisa dele

A /trend-hunting (e depois o /conteudo-funil e o /criativos-funil na Aula 2) raspam Reels, TikTok e perfis. Para isso precisam da chave do Apify. É grátis para começar: o plano Free dá US$ 5 de crédito por mês, sem cartão. Setup em 1 vez só — sem MCP, é só a chave:

1. Crie a conta grátis em console.apify.com/sign-up e copie o Personal API token (Settings → API & Integrations).
2. Cole o token no arquivo .env na raiz, na variável APIFY_API_TOKEN (APIFY_API_KEY também é aceito). Se não tiver .env, rode cp .env.example .env (no Windows: copy .env.example .env) — ou deixe o /comecar fazer isso por você.
3. Pronto. A coleta roda por API direta (a skill chama api.apify.com com a sua chave) — nada de instalar servidor nem reiniciar. O .env nunca vai para o GitHub (já está no .gitignore).

Skill 3 de 5

/trend-hunting

Identifica tendências emergentes nas fontes do nicho (X, Reels, TikTok, LinkedIn) antes da saturação e já gera variações de hook prontas para teste. Ao ativar, ela pergunta se você quer caçar tendências também a partir de concorrentes (cole os links dos perfis) — o tema do nicho é sempre varrido; concorrente é um acréscimo. Requer o Apify configurado acima.

trend-hunting [seu nicho]

Arquivos que saem (7):

  • trends-{nicho}-{data}.md/.html/.pdf — 3-5 padrões emergentes + 2-3 em alta, por timing (3 formatos)
  • variacoes-teste-{data}.md/.html/.pdf — hooks prontos com estrutura e CTA (3 formatos)
  • briefing-media-buyer.md — briefing de tráfego com orçamento e métrica de validação

Gate antes de avançar:

  • Pelo menos 3 padrões na fase "emergente"?
  • Cada variação tem hook + estrutura + CTA (não só o hook solto)?
  • O briefing do media buyer tem métrica de validação clara (CTR x%, CPL R$y)?
Skill 4 de 5

/swipe-file

Organiza os criativos vencedores que vieram do espião e do trend-hunting numa biblioteca categorizada por tipo, formato, padrão e nicho. Vira referência viva para a Copy e o Media Buyer. É inspiração, não cópia: você extrai o padrão e adapta ao seu ICP.

swipe-file capturar
swipe-file briefing [tipo]

Arquivos que saem (6):

  • swipe-file-index.md/.html/.pdf — índice master da biblioteca (3 formatos)
  • briefing-swipe-file.md/.html/.pdf — handoff para Copy e Media Buyer: top padrões + adaptações ao seu ICP (3 formatos)

Gate antes de avançar:

  • Capturou só criativos rodando há mais de 7 dias (com prova de performance)?
  • Cada criativo tem link da fonte (sem link → descarta)?
  • O briefing tem pelo menos 10 referências adaptadas ao seu ICP?
Skill 5 de 5

/offerbook

Consolida tudo num offerbook de 7 blocos (Materiais · Oferta · Avatar · História · História v2 · Análise de Cliente · Copy). É o brief de Story Selling que alimenta toda a copy depois — página, e-mails, ads. Avatar e Análise de Cliente vêm da pesquisa real; Oferta, História e Copy são escritas na voz da marca. Marca as lacunas (o que é decisão do dono vs. da equipe) e confere campo a campo antes de fechar.

offerbook [nome do seu produto]

Arquivos que saem (4):

  • briefing-offerbook.md — gate que precisa estar aprovado antes do offerbook
  • offerbook-{produto}.md — o Livro da Oferta (fonte de verdade)
  • offerbook-{produto}.docx — versão Word no Template Academia Lendária
  • offerbook-{produto}.html — renderizado, com TOC lateral

Pré-requisito do .docx: pip install python-docx (instala uma vez). No Windows, se pip não for reconhecido: py -m pip install python-docx.

Gate antes de fechar:

  • Todos os 7 blocos preenchidos?
  • Avatar e Análise de Cliente têm citações verbatim do lead?
  • Dono da oferta aprovou? (sem aprovação, não libera copy)

Parte 2 · Aula 2

Aula 2: Identidade → Funil completo

Com a pesquisa e a oferta prontas, você monta o funil de vendas ponta a ponta: identidade visual, página, e-mails, conteúdo e recuperação de quem não comprou. Cada peça é uma skill; você roda na ordem que o /metodo-funil mandar. Todo entregável sai em 3 formatos (.md, .html com a sua marca, .pdf).

1

Atualize o repo e rode /comecar de novo (rápido)

As skills da Aula 2 chegaram por atualização. Na pasta do cohort, rode git pull, abra o claude e rode /comecar mais uma vez — dessa vez é rápido: confere as skills novas, a chave do Apify e resolve a /design-md duplicada, se houver.

git pull
comecar
Onde tudo fica: uma pasta por projeto

Todo o trabalho de um nicho vive em projetos/{slug}/ — o slug é o nome do nicho em minúsculas com hifens (ex.: curso-de-ingles). Se a Aula 1 ainda estiver com arquivos soltos na raiz, a primeira skill da Aula 2 que precisar do projeto (/design-md ou /metodo-funil) migra o pack inteiro para projetos/{slug}/ antes de continuar — você não move nada à mão. Daqui pra frente cada skill só lê e grava dentro dessa pasta.

Passo 1 Identidade visual

/design-md — a identidade da sua marca

A primeira skill da Aula 2. Cria o DESIGN.md — cores, fontes, logo — que toda skill de HTML/PDF usa depois. Começa perguntando qual dos 5 caminhos: usar um design pronto · extrair de um site (URL) · criar do zero pelo seu nicho/avatar · a partir de referências (prints do Pinterest, mood board) · ou neutro. Mostra o preview, se autocritica e itera até o seu ok. Se a Aula 1 ainda estiver solta na raiz, ela migra o pack pra projetos/{slug}/; quando o projeto já existe, salva o DESIGN.md, tokens.json e preview.html direto no lugar certo.

/design-md

Você recebe: DESIGN.md + tokens.json + preview.html em projetos/{slug}/. Guia completo dos 5 caminhos: aula-02/aula-design-md-GUIA-ALUNO.html.

Passo 2 Gate obrigatório · o mapa

/metodo-funil — o diagnóstico e o mapa de execução

Não escolha o formato do funil você mesmo. Rode o /metodo-funil: ele lê o seu offerbook, faz até 3 perguntas (origem do público, ticket/formato, prova disponível), diagnostica o estágio de consciência (1 a 5) e devolve o mapa de execução — a lista ordenada das peças a construir, com a skill de cada uma. É o mapa que dita a ordem dali em diante.

/metodo-funil

Você recebe: estágio de consciência, funil prescrito, regras de anúncio por estágio e o mapa completo. Salvo em funil.md.

Negócio local? O WhatsApp assume o lugar do e-mail

Se o seu negócio é local (clínica, restaurante, serviço de bairro), o mapa troca o e-mail pelo WhatsApp: onde este guia diz /email-funil, você usa o /whatsapp-funil — mesmas etapas (confirmação, lembrete, carrinho abandonado, recuperação), no canal que o seu público realmente usa. O /metodo-funil te diz quando esse é o seu caso.

Do passo 3 em diante, siga o mapa. A ordem canônica das peças é esta:

/copy-funil
A fundação da mensagem. Pipeline em 3 fluxos (Lançamento Completo, Tráfego Pago, Otimização), com checkpoints de revisão. Gera headline (25 opções, 5 finalistas), bullets, VSL ou carta e a auditoria. É o cérebro do qual as outras peças puxam a copy.

Lê: offerbook.md. Gera: copy.md

/quiz · /webinario · /vsl · /advertorial · /lancamento
Só se o mapa mandar. O /metodo-funil prescreve o formato de meio/topo pelo estágio: quiz (nível 4), webinário (4 e 3), VSL e advertorial (5), lançamento (5 com lista). Rode a skill correspondente — não antes do mapa pedir.

Gera: quiz.md · webinario.md · vsl.md · advertorial.md · lancamento.md

/pagina-vendas-funil
A página de vendas com os 16 elementos topo a fundo (headline, VSL, mecanismo, stack, ancoragem, prova, bullets, bônus, garantia, escassez, FAQ, CTA…), aplicando os tokens do seu DESIGN.md. A headline passa em 3 testes antes de avançar.

Lê: copy.md + DESIGN.md. Gera: pagina/

/email-funil
E-mails HTML prontos pra colar na sua ferramenta de disparo, em 6 tipos (convite, confirmação, lembrete, recap, nurture, venda). Header, cor, botão e footer saem do seu DESIGN.md. Não dispara nada — você revisa antes. (Negócio local: use o /whatsapp-funil no lugar.)

Lê: offerbook.md + DESIGN.md. Gera: emails/

/conteudo-funil
Pergunta a plataforma (Instagram, YouTube, TikTok, X, LinkedIn) e o formato, coleta os conteúdos em alta do tema, você escolhe quais modelar e ela gera na sua voz em lote de 3 peças por nível do funil (topo, meio, fundo = 9). Carrosséis viram PNGs 1080x1350 na sua identidade. Você aprova a pauta antes.

Lê: offerbook.md + DESIGN.md. Gera: conteudo/ + carrossel/

/recuperacao-funil
A sequência de quem chegou no checkout e não comprou. Cada comportamento revela uma objeção: carrinho abandonado (leve), cartão recusado (urgente), boleto gerado (antes de vencer), downsell e re-elevação ao nível 4. Recuperar quem já chegou custa menos que adquirir um lead novo.

Lê: offerbook.md + funil.md. Gera: recuperacao.md

/backend-funil
Onde mora o lucro. Mapa ramificado (comprou → upsell 1 → upsell 2 / não comprou → downsell), janela de 3-7 segundos pós-checkout, janela de 4h da sequência, OTO de verdade e raciocínio de LTV. Rode quando o mapa incluir back-end.

Lê: offerbook.md + funil.md. Gera: back-end.md

/cro-funil
Fecha o ciclo: planilha de KPIs por etapa, identifica o gargalo, prioriza o teste A/B de maior alavancagem (headline primeiro), exige mínimo de 1.000 views únicos por teste e diz quando o funil está pronto pra escalar verba.

Lê: funil.md + copy.md. Gera: cro.md

A qualquer momento: /status-funil

Entre uma skill e outra, rode /status-funil. Ele lê a pasta projetos/{slug}/, marca [x] o que existe e [ ] o que falta, e aponta a primeira lacuna como "você está aqui". Não cria nem altera nada — só lê. É a bússola do funil.

O Book do Funil — o hub que reúne tudo

Toda peça que a Aula 2 gera aparece no Book do Funil: o index.html dentro de projetos/{slug}/. É o painel que reúne página, e-mails, conteúdo, recuperação e as pendências num só lugar — inclusive as amostras e checkpoints, que entram como cartão e galeria, nunca como arquivo solto no chat. Abra por ele para chegar em qualquer entregável:

macOS: open projetos/{slug}/index.html
Windows: start "" projetos\{slug}\index.html

As pendências (decisões do dono, dados que faltam) ficam listadas no próprio Book — é onde você vê num relance o que trava a próxima etapa.

Bônus opcionais

Fora do núcleo do funil, três skills reforçam a oferta quando fizer sentido:

/mockup-produto-funil
Prompts de geração de imagem parametrizados pelo seu DESIGN.md para cada item do offerbook: capa de ebook, tela de área de membros, box do produto, bundle de bônus. Você gera as imagens na ferramenta que quiser.

Gera: mockups/ (prompts)

/bonus-funil
Cria os entregáveis dos bônus de verdade (não só a lista): ebook, workbook, checklist, guia, template saem completos (md + HTML + PDF); áudio/vídeo viram roteiro pronto pra gravar. Fecha a promessa da oferta.

Gera: os bônus prometidos no offerbook

/criativos-funil
Espiona a Biblioteca de Anúncios do concorrente e vira criativos na sua voz: roteiros de vídeo e banners estáticos (HTML → PNG) na sua identidade, por rede e formato. Valida 1 amostra antes de gerar o lote.

Gera: criativos/ (roteiros + banners)

As regras de ouro do funil

1

Offerbook antes de tudo

Nenhuma skill da Aula 2 funciona sem o projetos/{slug}/offerbook.md. Se não existir, volte à Aula 1.

2

Rode na ordem do mapa

Cada skill usa o output da anterior. O /metodo-funil devolve o mapa — siga o mapa, não pule etapas.

3

DESIGN.md antes de qualquer HTML

Toda skill que gera HTML ou PDF precisa do DESIGN.md. Sem ele a skill para. Nenhuma cor ou fonte é inventada.

4

A headline é o teste mais importante

Não teste cor de botão. O que move conversão é headline (primeiro), depois oferta e preço, depois prova casada ao estágio. Mínimo de 1.000 views únicos por teste.

5

Prova casada ao estágio

Depoimento puro converte no nível 2 (consciente do produto). Para frio (5) ou morno (4-3) use estudo de caso ou narrativa. Casamento errado = campanha que não converte.

6

A skill estrutura, você publica

Nenhuma skill sobe página, dispara e-mail ou publica anúncio. Ela entrega a estrutura em 3 formatos — a revisão e a publicação final são suas.

Parte 3

Deu errado? (socorro)

Os travamentos mais comuns e a saída de cada um. Se nenhum resolver, a regra final está no fim.

Aparecem duas /design-md no menu

É a versão antiga da Aula 1 instalada globalmente brigando com a do projeto. Apague a antiga e reabra a sessão: no Mac/Linux, apague a pasta ~/.claude/skills/design-md; no Windows, apague C:\Users\SeuNome\.claude\skills\design-md. A do projeto (dentro de .claude/skills) é a que fica. O /comecar também resolve isso.

As skills não aparecem quando digito /

No Claude Code as skills ficam em .claude/skills e carregam sozinhas — confira que você abriu o claude dentro da pasta cohort-de-marketing (e rodou git pull para as skills novas da Aula 2). No Codex, .agents/skills já vem como espelho literal de .claude/skills. Se não aparecer, rode /comecar ou peça ao Codex para reespelhar a pasta e reabrir a sessão.

Apify: sem chave ou estourou a cota

Sem chave: a skill de coleta para e pede a configuração — crie a conta grátis em console.apify.com/sign-up, copie o Personal API token e cole no .env na variável APIFY_API_TOKEN (ou APIFY_API_KEY). O PDF não gera

Referência

Glossário — todos os termos

Não decore. Volte aqui sempre que esbarrar com um termo no resto do guia. Se você é de marketing, pode pular.

ICP (Ideal Customer Profile)

O perfil do cliente ideal — quem compra rápido, paga o preço cheio e indica pra outros. Não é "todo mundo que poderia comprar".

Avatar

A versão "personagem" do seu ICP — com nome, idade, dor real (citação literal), desejo, medo. A pessoa que você consegue imaginar conversando.

Verbatim

A frase literal do cliente, copiada como ele falou. É o que ancora avatar e copy — sem invenção. O que não tem verbatim é marcado [SUPOSIÇÃO].

Headline

A primeira frase que o cliente lê — no anúncio, na página, no e-mail. Define se ele continua ou fecha a aba. É o elemento nº 1 a testar.

Dossiê do concorrente

Relatório que mostra tudo o que o concorrente faz: anúncios, ofertas, ângulos e onde ele é fraco (as brechas). Serve pra você não inventar do zero.

Swipe file

Biblioteca de anúncios, headlines e copys que vendem bem no seu nicho. É referência pra extrair padrão, nunca pra copiar literal.

Offerbook (Livro da Oferta)

O brief mestre da oferta em 7 blocos. Tudo que copy, media buyer e designer leem antes de produzir. Sem offerbook aprovado, ninguém escreve copy.

Estágio de consciência (1 a 5)

O quanto o público conhece o problema e a solução: 5 inconsciente, 4 consciente do problema, 3 da solução, 2 do produto, 1 mais consciente. Decide o tipo de funil, anúncio e prova.

DESIGN.md

O arquivo com a identidade visual da sua marca (cores, fontes, logo). Toda skill que gera HTML ou PDF lê dele — nunca inventa cor nem fonte.

Slug / projeto

O nome curto do nicho em minúsculas com hifens (ex.: curso-de-ingles). Nomeia a pasta projetos/{slug}/ onde todo o funil daquele nicho vive.

Book do Funil

O index.html dentro de projetos/{slug}/: o hub que reúne todas as peças do funil e as pendências num painel só.

VSL (Video Sales Letter)

Carta de vendas em vídeo — o vídeo que apresenta dor, mecanismo e oferta. Funil típico de público frio (nível 5).

Upsell · OTO · Downsell

O back-end: upsell é a oferta que sobe o ticket de quem comprou; OTO (One Time Offer) é oferta única, sem repetir; downsell é a alternativa menor pra quem recusou.

CRO (otimização de conversão)

Medir cada etapa do funil por KPIs, achar o gargalo e testar (A/B) o que mais move resultado, antes de escalar verba.

Recuperação

A sequência para quem chegou no checkout e não comprou (carrinho abandonado, cartão recusado, boleto). Custa menos que adquirir lead novo.
