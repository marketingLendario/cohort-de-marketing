Parte A

Fundamentos absolutos

A.1 — O que é uma API e uma "chave"

API = a porta de serviço de um site: em vez de você clicar na tela, um programa entra por essa porta e pega os dados direto. Chave/token = a senha dessa porta — um texto comprido (ex.: apify_api_x7Kb9...). Quem tem a chave age em seu nome: trate como senha de banco. Escopo = o que a chave pode fazer (ex.: ads_read = só LER anúncios).

A.2 — As 4 regras de ouro

  1. A chave aparece UMA vez só na maioria dos sites (Meta, OpenAI, Anthropic, Figma). Copie na hora, cole no .env, salve. Perdeu? Revogue no site e gere outra — nunca "procure onde ficou".
  2. Nunca cole em lugar público (grupo, print, planilha). No cohort ela vive num único lugar: o .env na raiz.
  3. O .env nunca sobe pro GitHub (o .gitignore garante). Vazou? Revogue e gere outra.
  4. Colar errado é o erro nº 1. Sem espaço, sem aspas, tudo numa linha. Certo: APIFY_API_TOKEN=apify_api_abc123. Errado: APIFY_API_TOKEN = "apify_api_abc123".

A.3 — Criar e editar o .env (uma vez)

  1. Terminal na raiz do cohort-de-marketing (Windows: pasta aberta no Explorador → barra de endereço → digite cmd → Enter).
  2. Copie o modelo: Mac/Linux cp .env.example .env · Windows copy .env.example .env.
  3. Abra pra editar: Windows notepad .env · Mac open -e .env.
  4. Pegadinha do Bloco de Notas: salvar como tipo "Todos os arquivos", nome exatamente .env — se virar .env.txt, nada funciona (no Explorador, Exibir → marque "Extensões de nomes de arquivo" pra conferir).
  5. Jeito mais fácil: rode /comecar no Claude Code — ele cria o .env e cola as chaves com você.

A.4 — Conferir que salvou

Na raiz: grep -E "NOME_DA_VARIAVEL" .env (Windows/cmd: findstr "NOME_DA_VARIAVEL" .env). Tem que imprimir a linha. Não imprimiu = não salvou.

B.1 — APIFY (APIFY_API_TOKEN)

Serve para: espião, trend-hunting, conteúdo e criativos coletarem dados reais. · Custo: Free = US$ 5/mês de crédito, sem cartão. · Pré-requisito: um e-mail.
  1. Abra https://console.apify.com/sign-up.
  2. Crie a conta: "Sign up with Google" (mais fácil) ou e-mail + senha. Com e-mail, o Apify manda um link de verificação — clique nele antes de continuar.
  3. Questionário de boas-vindas? Responda qualquer coisa ou pule — não afeta nada.
  4. No Console, menu lateral esquerdo, desça até Settings (engrenagem, canto inferior esquerdo).
  5. Abra a aba "API & Integrations" (em alguns layouts: "Integrations" → "API tokens" — é a mesma tela).
  6. Copie o Personal API token (começa com apify_api_) pelo ícone de copiar ao lado.
  7. Cole em APIFY_API_TOKEN= no .env (logo após o =, sem espaço). Salve.
Teste de sucesso

grep -E "APIFY_API_(TOKEN|KEY)" .env imprime a linha; a próxima skill de coleta roda sem pedir a chave.

A skill avisa e cai no modo manual; renova mês que vem. · APIFY_API_KEY também é aceito, mas use o oficial. · Boa notícia (diferente da Meta/Hotmart): o token do Apify fica sempre visível em Settings — perdeu, volta lá e copia de novo; e não há aprovação nem espera.

B.3 — FUNDAÇÃO META: BM, Página, conta de anúncios e pagamento (do zero)

Isso nenhum doc do curso ensinava — está aqui completo. Sem esta fundação, nada da Aula 3/4 existe.

B.3.1 — Conta pessoal do Facebook

Perfil real e "aquecido" (foto, amigos, uso normal). Perfil novo só pra anunciar = alto risco de bloqueio automático. Se não tem: facebook.com → "Criar nova conta" → dados reais → confirmar e-mail/telefone → usar alguns dias antes de anunciar.

B.3.2 — Criar o Business Manager

  1. Logado no Facebook, abra https://business.facebook.com/overview.
  2. "Criar uma conta" (canto superior direito).
  3. Preencha nome do negócio, seu nome e e-mail comercial → Avançar.
  4. Confirme o e-mail que a Meta mandou.
  5. Guarde o endereço das configurações: business.facebook.com/settings — tudo a seguir mora aqui.

B.3.3 — Criar (ou vincular) a Página

  1. business.facebook.com/settings → Contas → Páginas → botão azul "Adicionar".
  2. "Criar uma nova Página" (nome = a marca, categoria do negócio) ou "Adicionar uma Página" existente.
  3. Capriche o básico (logo, capa, descrição) — Página vazia converte pior e levanta suspeita.

B.3.4 — Vincular o Instagram

  1. No app do Instagram: Configurações → Tipo de conta → Profissional (Empresa) — grátis.
  2. Business settings → Contas → Contas do Instagram → Adicionar → login do Instagram.
  3. Na Página do Facebook → Configurações → Instagram → conectar.

B.3.5 — Criar a conta de anúncios

  1. Business settings → Contas → Contas de anúncios → Adicionar → "Criar uma nova conta de anúncios".
  2. Nome + fuso (GMT-03:00) São Paulo + moeda BRL. ⚠️ Fuso e moeda não mudam depois — errou, cria outra.
  3. "Será usada para" → Meu negócio → Criar.
  4. Adicione você mesmo com Gerenciar conta de anúncios (acesso total).

B.3.6 — Forma de pagamento

  1. Business settings → Pagamentos → selecione a conta → "Adicionar forma de pagamento".
  2. Brasil: cartão de crédito (recomendado — pós-pago). Boleto/Pix (pré-pago, carrega saldo) existe, mas ⚠️ evite como forma principal — conta bloqueada com saldo pré-pago = dinheiro preso até a análise resolver (referência de mercado); detalhes no guia-meta-fundacao, passo 6.
  3. Confira: adsmanager.facebook.com → engrenagem → Configurações de pagamento → sem aviso vermelho.

B.3.7 — Verificar o domínio

  1. Tenha um domínio próprio (ex.: suamarca.com.br — registro.br, ~R$40/ano).
  2. Business settings → Segurança da marca → Domínios → Adicionar → digite o domínio.
  3. Método mais simples: verificação por DNS (TXT) — copie o código facebook-domain-verification=..., crie um registro TXT no painel do seu provedor de domínio, aguarde (minutos a horas), clique Verificar.
  4. Sem site próprio ainda? Pule e anote como pendência — o checkout Hotmart usa o domínio da Hotmart.

B.4 — META_ACCESS_TOKEN + App (o token principal, System User)

Serve para: zelador Modo API + painel da Aula 4. Só leitura (ads_read). · Pré-requisito: fundação B.3 completa. · Fontes no repo: aula-03/materiais/guia-app-meta-integracoes.html (visual) e aula-04/docs/tutorial-token-meta.md (texto).
Antes de começar — como a Meta se comporta

A Meta (a) muda nomes de menu sem aviso, (b) mostra telas diferentes pra contas diferentes (o mesmo clique pode abrir assistentes distintos) e (c) demora pra propagar permissões (minutos) e aprovar verificações (dias). Cada passo abaixo descreve o que a tela faz, as variações conhecidas e o tempo de espera. Tela não bate com nenhuma variação? Use a busca do menu lateral, ou o Prompt 3 do mapa-guiado — a IA acha o caminho com você.

B.4.1 — Criar a conta de desenvolvedor (primeira vez · 5–10 min)

  1. Abra developers.facebook.com já logado no Facebook — o MESMO perfil que é admin do seu BM (confira no canto superior direito).
  2. Clique em "Começar" / "Get Started". Se em vez disso você já vê "Meus apps" no topo: sua conta de dev já existe — pule pra B.4.2.
  3. O assistente tem 4 etapas, uma tela cada: Registrar (aceitar termos → Avançar) → Verificar conta (digite seu celular → chega um código de 6 dígitos por SMS → digite; não chegou em 2 min? "Update Mobile Number" reenvia) → Informações de contato (confirme o e-mail) → Sobre você (escolha algo como "profissional de marketing" — não trava nada depois).
  4. Prova de que deu certo: o menu "Meus apps" aparece no topo.

B.4.2 — Criar o app (10 min) — com as DUAS variações de tela

  1. "Meus apps" → botão verde "Criar app".
  2. Tela "Requisitos" (se aparecer): resumo do que você vai precisar ("um portfólio empresarial…") → Avançar.
  3. Tela de CASOS DE USO — cartões clicáveis, com duas variações:
    Variação A (mais comum hoje): cartões tipo "Autenticar e solicitar dados", "Interação de WhatsApp", "Gerenciar tudo na sua Página", "Outro". Para Marketing API/leitura de anúncios: selecione "Outro" → Avançar → "Selecione um tipo de app": "Negócios"/"Empresa" (Business) → Avançar.
    Variação B (contas antigas): vai direto a "Que tipo de app?" → escolha Empresa.
  4. Tela "Detalhes do app": nome ex. Cohort Trafego — sem acento, sem emoji, e ⚠️ nunca com "Facebook", "Meta" ou "Insta" no nome (recusa na hora). E-mail de contato: o seu.
  5. Campo "Portfólio empresarial" (Business Manager) — ⚠️ o passo mais importante e mais esquecido: selecione o BM da B.3. Deixou em branco? O app nasce solto, a Meta pode não enxergar sua BM depois e a lista de permissões do token vem VAZIA na B.4.5 — retrabalho garantido. Selecione AGORA.
  6. "Criar app" → a Meta pede sua senha do Facebook (ou código 2FA) → confirme.
  7. Você cai no painel do app. Duas coisas pra notar: o seletor "Modo do app: Desenvolvimento" no topo (é o modo certo — NÃO precisa publicar pra ler os SEUS dados; deixe assim) e, se aparecer o banner "Acesso à API restrito", clique "Ver ações necessárias" e siga (normalmente só confirmar contato) — libera o acesso básico.
  8. Configurações → Básico: copie o ID do app → META_APP_ID=; clique "Mostrar" na Chave Secreta (pede a senha de novo) → META_APP_SECRET=. Os demais campos (domínios, política, ícone) não são necessários pra leitura dos próprios dados — só entram se um dia pedir App Review.
  9. Marketing API (recomendado): menu "Adicionar produto" ou bloco "Adicionar casos de uso" no dashboard → se existir Marketing API → "Configurar" (só adicionar já basta). Não aparece? Siga em frente sem culpa — pro token de System User com ads_read, o produto não é pré-requisito; quem destrava os escopos é a B.4.4.
  10. Vincular o app ao BM (só se errou o passo 5): business.facebook.com/settings → Contas → Aplicativos → "Adicionar" → "Adicionar um ID do app" → cole o ID → adicionar.

B.4.3 — Criar o Usuário do Sistema (o "usuário robô" · 5 min)

  1. business.facebook.com/settings. ⚠️ Variação: se abrir o Meta Business Suite (layout novo com feed), clique na engrenagem no canto inferior esquerdo → "Configurações da empresa" — cai na mesma tela clássica.
  2. Menu esquerdo → Usuários → Usuários do sistema → "Adicionar".
  3. ⚠️ Primeiro usuário do sistema da conta? A Meta pode pedir aceite de termos ou dizer "disponível em breve" — BM recém-criado às vezes precisa de algumas horas/dias de "idade" ou da Verificação de Empresa pra liberar. Aguarde e tente de novo.
  4. Nome leitor-trafego → função Funcionário (Admin não é preciso pra só ler; menos poder = menos risco) → Criar usuário do sistema.

B.4.4 — ATRIBUIR OS ATIVOS ⚠️ (ordem OBRIGATÓRIA: ativos ANTES do token)

  1. Com leitor-trafego selecionado → "Atribuir ativos" (em alguns layouts fica atrás do menu "..." Mais opções, no topo à direita).
  2. No modal, aba Apps → marque Cohort Trafego → ative a chavinha "Gerenciar app" (controle total) → "Salvar alterações". ← é ESTE toggle que destrava os escopos do token; sem ele a lista de permissões vem VAZIA — o erro nº 1 de todo mundo.
  3. Repita "Atribuir ativos" → aba Contas de anúncios → sua conta → "Ver desempenho" (parcial; NÃO precisa "Gerenciar campanhas") → Salvar.
  4. (Só p/ orgânico da Aula 4) repita → aba Páginas → sua Página → "Ver desempenho da Página" → Salvar.
  5. ⏱ Espere 2–5 minutos — a atribuição propaga com atraso; token gerado no segundo seguinte às vezes vem sem permissões.

B.4.5 — Gerar o token (2 min + atenção total ao copiar)

  1. Usuário selecionado → "Gerar novo token" (ou "..." → "Gerar token"). Abre o assistente.
  2. Tela "Selecionar app": escolha Cohort Trafego → Avançar.
  3. Expiração: opções típicas "60 dias" e "Nunca". Escolha "Nunca" se aparecer (permanente = sem refazer a cada 2 meses); só tem "60 dias"? Ok — anote no calendário pra regenerar.
  4. Permissões: marque somente ads_read ✅ + business_management ✅. Nada de escopo de escrita — o método é leitura. ⚠️ Lista vazia? App não atribuído (B.4.4-2) ou propagação pendente (B.4.4-5).
  5. "Gerar token" → aparece UMA ÚNICA VEZ → copie AGORA → META_ACCESS_TOKEN= no .env → salve → só então feche. Fechou sem copiar? "Gerar novo token" de novo (o anterior segue válido, mas irrecuperável).

B.4.6 — IDs automáticos

  1. node scripts/zelador-audit.mjs --gravar-env — descobre e grava conta, pixel, página e BM (o Instagram você pega na mão, B.4.7).
  2. Validar sem alterar: node scripts/zelador-audit.mjs --json.
  3. Mais de uma conta/pixel/página? O --gravar-env não grava nada nesse caso (nunca escolhe por você): o relatório lista as opções pra você copiar a linha certa no .env. Alternativa só p/ contas: painel-trafego-data.mjs --listar-contas.

B.4.6 — Plano B: cada ID na mão

  • Conta de anúncios: adsmanager.facebook.com → número na URL (act=123...) → grave META_AD_ACCOUNT_ID=123... — só o número, sem o act_ (padrão do repo; com prefixo também funciona).
  • Pixel: Events Manager (business.facebook.com/events_manager2) → Fontes de dados → número embaixo do nome.
  • Página: Página → "Sobre" → "ID da Página" (ou business settings → Contas → Páginas).
  • BM: business settings → Informações da empresa → "ID do Gerenciador de Negócios".
  • Instagram: business settings → Contas → Contas do Instagram → número embaixo do @.

B.5 — META_AD_LIBRARY_TOKEN (espião automático — opcional)

Sem ele a skill funciona igual: abre a Biblioteca pública (facebook.com/ads/library) e você cola os textos. O pré-requisito é pesado — por isso deixe por último.
  1. Confirmar a identidade (uma vez na vida · 1–3 dias úteis): abra facebook.com/ID logado. O fluxo pede país de residência → upload de documento oficial com foto (RG, CNH ou passaporte — nítido, sem corte, sem reflexo). Envie e aguarde a resposta por notificação/e-mail em 1 a 3 dias úteis. ⚠️ Não conclui ao vivo numa aula — dispare antes.
  2. Aceitar os termos: aprovada a identidade, facebook.com/ads/library/api/ → "Get Started" → leia e aceite. A página libera as seções de acesso.
  3. App: o mesmo Cohort Trafego da B.4 serve. Se a página oferecer "adicionar o produto Ad Library API" ao app, adicione.
  4. Gerar o token — 2 caminhos equivalentes: na seção "Tools"/"Ferramentas" da própria página; OU no Graph API Explorer (developers.facebook.com/tools/explorer): painel direito → selecione o app → "Generate Access Token" → confirmar login → o token aparece no campo. Sem escopo especial.
  5. Cole em META_AD_LIBRARY_TOKEN=.
  6. Teste de sucesso (30 s): no Graph API Explorer, consulte ads_archive?search_terms='emagrecimento'&ad_reached_countries=['BR'] → Enviar. Lista JSON no meio da tela = funcionando.
Se der errado

"Application does not have permission" → termos não aceitos com ESTE perfil, ou identidade ainda em análise. · Token expira em horas → estenda no Access Token Debugger (developers.facebook.com/tools/debug/accesstoken → "Estender", vira ~60 dias). · Na dúvida: o fallback manual (abrir a Biblioteca pública e colar os textos) dá o mesmo resultado analítico, sem pré-requisito nenhum.

B.6 — Orgânico: META_PAGE_ACCESS_TOKEN e META_INSTAGRAM_TOKEN (avançado, opcional)

Sem eles o painel funciona — a aba Engajamento fica "em breve". Caminho A = modo desenvolvedor (sem App Review), validade ~60 dias, só na SUA conta.

Facebook (META_PAGE_ACCESS_TOKEN)

  1. Pré: admin do app E da Página; no app → Casos de uso → "Gerenciar tudo na sua Página" → Personalizar → adicione pages_read_engagement + pages_read_user_content.
  2. Graph API Explorer (developers.facebook.com/tools/explorer) → seu app → "Get User Access Token" → marque pages_read_engagement, pages_read_user_content, pages_show_list → Generate.
  3. Estenda: Access Token Debugger (developers.facebook.com/tools/debug/accesstoken) → cole → "Estender token" (~60 dias) → copie o longo.
  4. META_PAGE_ACCESS_TOKEN= → teste node scripts/painel-trafego-data.mjs --account --perfil → perfil.disponivel: true.
  5. ⚠️ Em produção (qualquer conta), posts/comentários exigem App Review aprovado (semanas) — limitação real da Meta.

Instagram (META_INSTAGRAM_TOKEN — API nova, "login do Instagram")

  1. É o seu caso se o app mostra escopos instagram_business_basic/instagram_business_manage_comments.
  2. App → Casos de uso → API do Instagram → Personalizar → aba Funções → adicione seu @ como "Testador do Instagram".
  3. Aceite o convite DENTRO do Instagram: Configurações → "Apps e sites" (ou a notificação). ← passo esquecido = causa nº 1 de falha.
  4. De volta ao app → "Gerar tokens de acesso" → Adicionar conta → selecione o @ → "Add all required permissions" → gerar (já sai longo).
  5. META_INSTAGRAM_TOKEN= → teste com --perfil. Pré: conta Instagram Profissional vinculada à Página (B.3.4).

B.7 — HOTMART (HOTMART_CLIENT_ID + HOTMART_CLIENT_SECRET)

🔀 Esta seção virou guia próprio: guias/03-conexoes-e-apis/guia-hotmart.md (erros M1–M7). O pré-requisito dela (produto com checkout) ganhou o guia-produto-e-checkout.md.

Serve para: a Aula 4 ler as vendas reais (caixa) — o ROAS rastreado vira "Real". · Pré-requisito: conta de Produtor — a área de credenciais normalmente é exclusiva de Produtor; o menu não aparece? Sua conta deve ser só de Afiliado — pule (o painel funciona sem).
  1. Login em app.hotmart.com com a conta de Produtor (a dona do produto — afiliado não vê este menu).
  2. Menu lateral → Ferramentas. ⚠️ Variações: o item pode se chamar só "Credenciais" ou "Credenciais de Desenvolvedor". Não achou? Lupa de busca do topo → "credenciais".

Teste de sucesso

node scripts/hotmart-vendas.mjs --dias=90 --debug responde (mesmo "0 vendas" prova que conectou).

O truque do ROAS "Real": o parâmetro sck

No link do anúncio que aponta pro checkout, inclua o identificador da campanha no sck. Exemplo literal: https://pay.hotmart.com/A12345678B?sck=meta-c1-frio. Regras (fonte: manual-trackeamento da aula-03): máx. 30 caracteres, só letras, números, hífen e pipe — underscore é proibido (o sanitizador converte _ em -). A venda volta no webhook com esse sck e o painel casa venda ↔ campanha ("Real"). Sem sck: nome do produto ou janela temporal ("Estimado").

Se der errado

401 = ID/Secret errados ou credencial revogada (gere outra). · "CLIENT_ID ausente" = confira o nome exato da linha no .env.

Parte C

Trackeamento sem programar (Pixel + CAPI via Hotmart)

🔀 Esta parte virou guia próprio: guias/03-conexoes-e-apis/guia-pixel-capi.md (erros T1–T9, com o caminho genérico WordPress/GTM). O conteúdo abaixo permanece como referência dentro do índice.

O manual-trackeamento.html da aula-03 é o blueprint avançado (pra quem tem programador). Este é o caminho do aluno comum vendendo via Hotmart — só cliques, zero código.

C.1 — Criar o Pixel ("Conjunto de dados")

  1. business.facebook.com/events_manager2 → "Conectar dados" (ou botão verde "Conectar fontes de dados").
  2. "Web" → Conectar → nome (ex.: Pixel Sua Marca) → Criar.
  3. Na pergunta de instalação, pode fechar — a Hotmart instala (C.2). Anote o ID (número embaixo do nome) = META_PIXEL_ID.

C.2 — Ligar Pixel + CAPI pela Hotmart (o pulo do gato)

  1. app.hotmart.com → Ferramentas → Pixel de rastreamento → escolha o produto → Facebook/Meta.
  2. Cole o ID do Pixel (C.1).
  3. Modo de envio: "Conversions API" ou "Ambos". O token da CAPI é gerado na Meta: Events Manager → seu pixel → aba Configurações → "API de Conversões" → "Gerar token de acesso" → cole no campo da Hotmart. Se a tela perguntar ONDE disparar: marque "visita à página de pagamento" (é o InitiateCheckout) e NÃO marque a página de produto da Hotmart — só infla evento sem valor (referência de mercado).
  4. Salve. A Hotmart dispara InitiateCheckout e Purchase com event_id automático — é isso que garante a deduplicação (navegador + servidor contam UMA vez).
  5. ⚠️ Boleto/Pix: na hora sai Payment Generated; o Purchase só dispara quando aprova — e só via CAPI. Sem CAPI, venda de boleto "some" do pixel.

C.2.5 — Plugar o pixel na SUA página de vendas (o passo que todo mundo pula)

  1. A Hotmart instala o pixel só no checkout. Na página de vendas é você que pluga — e as páginas geradas pelas skills já nascem pixel-ready.
  2. Abra o HTML da sua página (projetos/{slug}/pagina/...) → no <head>, o snippet do Meta Pixel vem pronto porém COMENTADO, com o placeholder [PLUG: SEU_PIXEL_ID].
  3. Substitua o placeholder pelo ID do pixel (C.1) e descomente o snippet (e o <noscript> logo abaixo). Republique.
  4. Só então o teste C.3 mostra o PageView.

C.3 — Testar que dispara (5 min)

  1. Instale a extensão Meta Pixel Helper (Chrome Web Store).
  2. Abra sua página de vendas → o ícone acende e lista PageView.
  3. Clique no botão de compra até o checkout → o Helper (ou Events Manager → "Testar eventos") lista InitiateCheckout.
  4. Events Manager → Visão geral: eventos aparecem no gráfico nas próximas horas (Purchase só com venda real).
  5. Deduplicação ok = evento "Navegador + Servidor" com selo verde de desduplicado.
⚠️ Duas manhas do Events Manager (referência de mercado)

(1) A Meta às vezes desativa/reativa sozinha configurações como a "Correspondência avançada automática" — depois de salvar qualquer coisa, reabra a tela e confirme que ficou como você deixou. (2) Alerta vermelho/amarelo no Events Manager raramente é crítico — leia o texto antes do pânico: "Purchase não dispara" quase sempre é problema de checkout/CAPI, não pixel "estragado" ("pixel frio/sujo" é mito).

C.4 — O gabarito do /zelador

BM ativo · conta ativa · Pixel disparando · CAPI ativa · Purchase deduplicado (event_id) · domínio verificado (B.3.7) · pagamento ok. Item crítico vermelho (todos, exceto o domínio) = o Estruturador não deixa subir campanha; domínio não verificado = status PARCIAL, que não bloqueia (siga com a pendência anotada). Dois itens a API não prova — deduplicação e domínio: mesmo no Modo API, o zelador confirma com VOCÊ na tela. Vai publicar via API (Gates 2/3)? Rode antes node scripts/zelador-audit.mjs --testar-escrita — é o que libera o Modo API do estruturador.

D.3 — Criando no Gerenciador de Anúncios, tela a tela

Nível campanha

  1. adsmanager.facebook.com → confira no topo a conta de anúncios certa.
  2. Botão verde "+ Criar".
  3. Objetivo: "Vendas" (checkout) ou "Cadastros" (lead) → Continuar. Se oferecer "configuração recomendada vs manual", escolha manual.
  4. Nome rastreável: [SUAMARCA] Oferta X - Vendas - 2026-07.
  5. Orçamento no CONJUNTO de anúncios (R$ 30,00/dia) — é como o Modo API do estruturador cria; se o Gerenciador oferecer "orçamento da campanha (Advantage/CBO)", desligue. Com 1 conjunto, o efeito é o mesmo.
  6. Categoria especial: só crédito/emprego/moradia/social/eleitoral — senão, em branco. → Avançar.

Nível conjunto de anúncios

  1. Nome: Frio - Advantage+ - BR.
  2. Conversão: local = Site · Pixel = o seu · Evento = Comprar (Purchase) (ou Lead). ⚠️ Evento cinza/"inativo" = o pixel nunca o registrou → volte ao C.3.
  3. Datas: início hoje · término em 7 dias (o default tem fim programado).
  4. Público: aceite o "Público Advantage+". Localização = Brasil (ou sua região) · idade no padrão/amplo (o default sagrado não restringe idade) · não empilhe interesses — no máx. 1 guarda-chuva, ou nenhum (100% amplo).
  5. Posicionamentos Advantage+ (automático) — não desmarque nada. → Avançar.

Nível anúncio (repita p/ cada finalista)

  1. Nome: H1 - dor - reels (hook + ângulo + formato).
  2. Identidade: sua Página + Instagram vinculado.
  3. "Criar anúncio" → formato "Imagem ou vídeo único".
  4. Adicionar mídia → suba o criativo (vídeo 9:16; revise os recortes dos outros posicionamentos no preview à direita).
  5. Texto principal = copy do finalista · Título = hook curto · Descrição opcional.
  6. CTA: "Saiba mais" (página de vendas) ou "Comprar agora" (checkout direto).
  7. URL do site já com rastreio: UTMs na página, ou sck no checkout — ex.: https://pay.hotmart.com/A12345678B?sck=meta-c1-frio.
  8. Confira o preview em Feed, Stories e Reels — nada cortado/ilegível.
  9. 2º e 3º criativos: menu esquerdo do editor → "Duplicar" o anúncio → troque só mídia/copy → renomeie.

Publicar

  1. Botão verde "Publicar" → entra "Em análise" (minutos a ~24 h).
  2. Recomendação do método: publique e pause imediatamente (toggle azul ao lado do nome) — revise com calma e ative quando estiver 100%. É o que o /estruturador faz no Modo API (Gate 2 cria pausado; Gate 3 ativa só com a sua ordem).
  3. Registre no PAINEL-DA-SEMANA.yaml: data, verba, criativos, critério de sucesso (ex.: ≥ 1 venda ou CPA ≤ R$ X em 7 dias) e critério de reversão (ex.: R$ 210 gastos sem venda = pausa).

D.4 — Os 7 dias e o ciclo semanal

  1. Dias 1–7: não mexa em NADA. Nem copy, nem público, nem "ajustadinha" — edição reseta o aprendizado. Única exceção: o circuit breaker (D.2).
  2. Dia 7: /leitor-de-metricas (com chaves B.4, lê sozinho; sem, você cola os números). Toda métrica com selo Real/Estimado/não fornecido.
  3. /diagnosticador: aponta UMA alavanca (hipótese + sucesso + reversão). Você decide e registra no PAINEL-DA-SEMANA.yaml (histórico — nunca apague semanas).
  4. Aula 4: /gestor-de-campanhas compara planejado × realizado (7/30 dias); /retroalimentacao devolve os aprendizados pro avatar e pra copy.
