# Conexao com a Aula 02 — Arquiteto de Funil

A Aula 02 (Arquiteto de Funil) **assume** que voce concluiu a Aula 01 e tem em maos:

1. ICP completo (`icp-{negocio}.md`)
2. Dossie de concorrentes (`competidores-{nicho}.md`)
3. Offerbook preenchido (`offerbook-{produto}.md`)
4. Swipe file com 10+ criativos winners categorizados

Sem esses 4 artefatos, a Aula 02 nao tem matéria-prima para trabalhar.

## O que a Aula 02 faz com cada artefato

| Artefato da Aula 01 | Uso na Aula 02 |
|---|---|
| ICP | Define publico-alvo dos anuncios + segmentacao + linguagem das paginas |
| Dossie de concorrentes | Posicionamento da oferta vs concorrentes + brecha competitiva |
| Offerbook | Estrutura da pagina de vendas (Bloco 4 e 5 do offerbook viram a VSL/sales page) |
| Swipe file | Referencias visuais para criativos (Reels, carousels, single image) |

## O que o Arquiteto de Funil entrega (Aula 02)

- Funil mapeado em etapas (topo, meio, fundo, pos)
- Pagina de vendas (LP ou VSL) baseada no Offerbook
- Bateria de 5-10 criativos prontos para rodar
- Estrategia de email marketing
- Setup de tracking (pixel, GTM, GA4)

## O que ele NAO faz (Aula 03 e 04 cuidam)

- Rodar campanha (Aula 03 - Media Buyer)
- Otimizar budget (Aula 03)
- Construir dashboard de receita (Aula 04 - Analise de Dados)

## Handoff (passagem de bastao da Aula 01 para Aula 02)

Voce entrega para o Arquiteto de Funil:

```
pacote-aula-01/
├── icp-{negocio}.md
├── icp-{negocio}-resumo.md
├── icp-{negocio}.html
├── competidores-{nicho}.md
├── trends-{nicho}-{data}.md
├── briefing-swipe-file.md
├── briefing-offerbook.md
├── offerbook-{produto}.md
├── swipe-file/
│   ├── ads-meta/
│   ├── posts-organicos/
│   ├── hooks/
│   └── swipe-file-index.md
└── lacunas-aula-01.md     <- importante: o que ficou pendente
```

## Checklist de handoff

Antes de passar para a Aula 02, validar:

- [ ] ICP validado com Modulo 10 (citacoes literais, N e %)
- [ ] Offerbook aprovado pelo dono do negocio
- [ ] Swipe file com minimo 10 criativos categorizados
- [ ] Brecha competitiva identificada e defensavel
- [ ] Lacunas listadas em documento separado

Se algum item nao passou, **nao avance** para a Aula 02. O Arquiteto de Funil nao tem como construir paginas com input ruim.
