Mapa Guiado do Projeto

O mapa completo do repositório, do zero ao avançado: o que ele é, como instalar tudo (com prompts prontos), as 37 skills em 4 aulas, as telas de navegação, o fluxo de dados de ponta a ponta, a anatomia do repo e o checklist interativo que prova que tudo funciona. Seção 0

O que é este repositório

Em uma frase: a fábrica de marketing do Cohort — 37 skills (roteiros de IA que executam trabalhos inteiros de funil: pesquisa, oferta, copy, design, páginas, e-mails, criativos, tráfego e análise de dados), organizadas em 4 aulas, operadas por CLI (Claude Code ou Codex) com aprovação humana em cada passo.

O que mudou em relação ao Ads Studio antigo

Não existe mais o painel web (npm run studio, Supabase, Docker, BFF) — o modo de uso é CLI puro. Em troca, o que era estático virou vivo: o mapa de skills é gerado do catálogo, o progresso é lido do disco (/status-funil + motor de prontidão), cada projeto tem um hub próprio (o Book do Funil) e a Aula 4 gera um painel de dados interativo que abre offline. Ganhou-se também a Aula 4 inteira (Central de Dados, +6 skills) e as integrações Meta Graph API (leitura) e Hotmart (caixa real).

Os 5 axiomas

1A IA executa, você aprova — nenhuma skill publica anúncio, dispara e-mail ou sobe página sozinha. O squad de tráfego é recommend-only; o /estruturador só publica via API em 3 gates, com a sua confirmação explícita.
2O estado são os arquivos (projetos/{slug}/), não o chat. Fechar ou limpar a conversa não perde nada — pra voltar, abra a IA na pasta e rode a próxima skill.
3Fonte única para cada coisa: oferta = offerbook.md · mensagem = copy.md · identidade = DESIGN.md · catálogo = data/skill-catalog.json · prontidão = skill-surface-contract.js + scripts/lib/skill-readiness.mjs.
4Dado sem fonte não existe: pesquisa exige citação literal (verbatim) ou marca [SUPOSIÇÃO]; métrica carrega selo Real / Estimado / Calculado / não fornecido — todo ROAS é Estimado (atribuição ≠ caixa); ausente nunca vira zero.
5O ciclo fecha — método O.F.T.R.: Oferta → Funil → Tráfego → Retroalimentação. Os dados reais da Aula 4 voltam pro avatar (Aula 1) e pra copy/offerbook (Aula 2), orientando a próxima rodada.

Aulas ≠ trabalho — abra sempre na raiz

As pastas aula-01/ a aula-04/ são material didático (guias, docs, templates, exemplos). O seu trabalho nunca é gravado dentro delas — vive em projetos/{slug}/, na raiz, criado automaticamente pela primeira skill que gravar algo. As skills só existem na raiz (.claude/skills/); abrir a IA dentro de aula-XX/ = skills não carregam e caminhos erram. Não importa qual aula você cursa: abre na raiz, roda a skill, tudo converge pro mesmo projetos/{slug}/.

Instruções para a IA que operar este repo

Se você é um assistente de IA operando neste repositório, siga estas regras ao usar este guia:

1Fontes de verdade: este arquivo é um mapa, não a fonte. Canônicos: README.md, AGENTS.md, .claude/skills/{nome}/SKILL.md, data/skill-catalog.json, data/skill-unlock-rules.json, skill-surface-contract.js, data/contracts/. Em conflito, a fonte vence este guia.
2Nunca instale nada sem perguntar antes. Ofereça a instalação em 1 linha, aguarde o "sim".
3Nunca deixe o usuário num beco (regra _shared/nunca-travar.md): todo erro vira explicação em português simples + UM próximo passo.
4O estado são os arquivos em projetos/{slug}/, nunca o chat. Nunca diga que fechar a conversa perde progresso.
5Nada é publicado sem aprovação humana explícita. Publicar, pausar ou alterar campanha na Meta é sempre ação humana.
6Toda métrica carrega selo: Real (literal da fonte) · Estimado (atribuição/projeção — todo ROAS) · Calculado (variação entre dois Real) · não fornecido (nunca inventar nem derivar).
7.claude/skills/ é a fonte canônica; .agents/skills/ é espelho byte a byte. Nunca deixe divergirem (cp -R .claude/skills/. .agents/skills/ refaz).
8projetos/ e .env não são versionados (gitignore). Nunca commite .env nem dados reais da conta do aluno.
9Slug de projeto: minúsculas, números e hífen (^[a-z0-9][a-z0-9-]*$).
10Pro usuário leigo: traduza jargão na primeira ocorrência (4 palavras entre parênteses); não infantilize o avançado.
Seção 1

Instalação

Trilha ZERO — prompts prontos (a IA instala; sempre perguntando antes)

Prompt 0 (oficial): /comecar (Codex: @comecar) — detecta o SO, confere git/Node/Python/Apify, instala o que faltar perguntando, aponta UM próximo comando. Rede de segurança do curso: travou em qualquer aula, rode de novo. Os prompts abaixo cobrem os casos específicos — clique em copiar e cole no chat da IA:

Prompt 1 — ambiente das skills
Leia o arquivo README.md e a skill .claude/skills/comecar/SKILL.md deste repositório. Depois verifique o que já tenho instalado (git, Node, Python) e se existe a chave do Apify no arquivo .env da raiz. Para cada coisa que faltar: me explique em 1 frase o que é e para que serve, me pergunte antes de instalar, e instale pelo caminho mais simples do meu sistema operacional. Não sou programador — traduza qualquer erro em português simples e nunca me deixe sem um próximo passo. Ao final, me diga qual é o único próximo comando que devo rodar.
Prompt 2 — Apify do zero
Ainda não tenho conta nem chave do Apify. Me guie passo a passo, com calma: (1) criar a conta gratuita em console.apify.com; (2) encontrar o Personal API token nas configurações; (3) eu colo o token aqui no chat e VOCÊ cria ou atualiza o arquivo .env na raiz deste projeto com a linha APIFY_API_TOKEN=... (o arquivo .env pode não existir ainda — crie se for o caso); (4) confirme que ficou salvo mostrando o resultado de um grep. Nunca invente o valor do token: use exatamente o que eu colar.
Prompt 3 — chaves da Meta (Aulas 3 e 4)
Quero conectar a minha conta de anúncios da Meta em modo LEITURA. Leia aula-04/docs/configurar-chaves-meta.md e aula-04/docs/tutorial-token-meta.md e me guie passo a passo: criar o app, gerar o token de usuário do sistema com business_management + ads_read, e preencher no .env as variáveis META_ACCESS_TOKEN e META_AD_ACCOUNT_ID. Se eu só tiver o token, rode node scripts/zelador-audit.mjs para descobrir os IDs pela API. Nada de publicar ou alterar campanha — só leitura. Se eu não quiser conectar agora, confirme que tudo roda em Modo Exemplo.
Prompt 4 — Python da Creative Factory
Quero usar a ads-creative-factory. Instale as dependências Python dela a partir de .claude/skills/ads-creative-factory/requirements.txt (me pergunte antes de instalar), rode o script doctor.py da mesma skill para diagnosticar o ambiente e me traduza o resultado: o que está ok e o que falta.
Prompt 5 — socorro (qualquer erro)
O comando que acabei de rodar deu erro. Leia a mensagem de erro acima, me explique em português simples o que aconteceu (sem jargão), me diga se é grave, e me proponha UMA correção de cada vez, perguntando antes de executar. Se a correção envolver apagar ou sobrescrever qualquer coisa, me avise explicitamente antes.

Passo a passo — Modo Aluno (o único modo)

  1. Obter o projeto (repo privado — o git pede login do GitHub; a "senha" no terminal é um Personal Access Token, não a senha comum; alternativa: Code → Download ZIP, mas sem git pull depois):
    git clone https://github.com/marketingLendario/cohort-de-marketing.git → cd cohort-de-marketing. Depois, sempre git pull (nunca clonar de novo por cima).
  2. Abrir a IA na pasta RAIZ (nunca dentro de aula-XX/): claude (ou codex). Digitar / + "funil" deve listar as skills.
  3. Rodar /comecar — fecha apontando UM próximo comando.
  4. Node (node --version, v18+) e Python (python --version; Windows: py --version; instalar do python.org marcando "Add Python to PATH"; digitar python sem ter instalado abre a loja da Microsoft = stub, não erro).
  5. Apify e o .env (não vem no clone, de propósito): conta em console.apify.com → Settings → API & Integrations → copiar o Personal API token (apify_api_...) → cp .env.example .env (Windows: copy) → colar em APIFY_API_TOKEN=. Conferir: grep -E "APIFY_API_(TOKEN|KEY)" .env. Bloco de Notas: salvar como "Todos os arquivos" com nome .env (senão vira .env.txt). Sem a chave o começo do funil funciona; a primeira skill de coleta configura com você — nunca "pular Apify" em silêncio.
  6. Chaves da Meta (Aula 3/4) e Hotmart (opcional, Aula 4) — guias: aula-04/docs/configurar-chaves-meta.md, tutorial-token-meta.md, tutorial-hotmart.md, e o passo a passo pra novato completo no SUPER GUIA de APIs. Sem elas, /zelador roda em Modo Manual e a Aula 4 em Modo Exemplo — nada trava.
  7. Creative Factory (quando for gerar criativos): python -m pip install -r .claude/skills/ads-creative-factory/requirements.txt e diagnóstico python .claude/skills/ads-creative-factory/scripts/doctor.py --json.

Fundações (glossário mínimo)

Terminal
Janela de comandos. Tudo aqui é comando, mas o /comecar guia cada um.
.env
Arquivo de chaves locais na raiz; não vem no clone (de propósito); nunca vai pro git.
Git
Baixa/atualiza o material do curso preservando projetos/.
Claude Code / Codex
Assistentes que executam as skills. .claude/skills/ (canônica) ↔ .agents/skills/ (espelho byte a byte pro Codex).
Node / npm
Motor da design-md (18+), dos validadores e dos scripts das Aulas 3/4 (zelador-audit, leitor, gestor, painel de dados).
Python
Coletores de pesquisa, DOCX do offerbook, Creative Factory (numpy/Pillow/PyYAML).
Chrome/Edge headless
Converte HTML→PDF (gerar_pdf.sh) e HTML→PNG (gerar_png.sh). Fallback: abrir o .html e imprimir.
Apify
Scraping com dados reais via API REST direta pelo .env (NÃO é MCP). Central nas skills de coleta.
Graph API (Meta)
Leitura da conta de anúncios: insights, campanhas, demografia, funil. Sempre read-only — exceto o /estruturador, que cria campanha pausada em 3 gates com a sua confirmação.
Hotmart API
Caixa real (vendas) pra atribuição honesta da Aula 4 — sem dado pessoal de comprador. ROAS só vira "Real" com venda conferida no caixa.
Contratos (JSON Schema)
data/contracts/ versiona as interfaces (project-brief, campaign-plan, weekly-panel, weekly-ledger, traffic-analytics, decision-outcome, source-reconciliation). skill-surface-contract.js valida catálogo, regras, brief e índice.
ArtifactIndex v1
Índice determinístico do que existe em projetos/{slug}/ (scripts/project-artifact-index.mjs): sha256, hardening contra path escape/symlink/segredos. A ponte entre o disco e o motor.
Motor de prontidão
scripts/lib/skill-readiness.mjs decide a próxima skill e a razão — usado pelo /status-funil e pelo mapa. Nunca hardcoded.
Testes e validadores
Bateria .test.mjs em scripts/ (release gate Aula 4, weekly ledger, circuit breaker, contratos, artifact index) + validate-mapa-*.mjs, validate-skill-catalog.mjs, verify-skill-contracts.mjs.
